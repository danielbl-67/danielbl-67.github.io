⚠️ Troubleshooting Git: Soluciones a Errores Clásicos

Escenario / Error ¿Por qué pasa? Solución / Comando
Merge Conflict:
Automatic merge failed		 Dos ramas modificaron la misma línea en el mismo archivo y Git no sabe cuál elegir. Reparar Abre el archivo, limpia las marcas <<<<<<< y haz un nuevo commit: git add . git commit -m "Fix merge conflict"
Oops, el commit:
Mensaje erróneo o incompleto		 Escribiste mal el mensaje del último commit local o te olvidaste de añadir un archivo. Modificar último git commit --amend -m "Mensaje corregido"
¡Pesa demasiado!
Fallo en el Push por archivo gigante		 Añadiste una base de datos .sql pesada o la carpeta node_modules sin querer. Sacar de Staging Quita el archivo manteniendo tus cambios intactos: git rm --cached [nombre_archivo] ¡No olvides meterlo luego en tu .gitignore!

✏️ Gestión de Cambios

Estado y Staging:

git status git add . git reset [archivo] # Unstage

Commits:

git commit -m "Mensaje" git commit --amend --no-edit # Corregir último sin cambiar texto

🌱 Ramas (Branches)

git branch -a # Ver todas git checkout -b [nombre] # Crear y saltar git switch [nombre] # Cambiar rama git merge [nombre] # Fusionar git branch -d [nombre] # Borrar local

🕒 Historial y Reset

git log --oneline --graph --all git reset --hard [id_commit] git stash # Guardar cambios temporalmente git stash pop # Recuperar stash

🌐 Remotos y Forking

git remote add origin git@github.com:danielbl-67/[repo].git git push origin [rama]

Trabajando con Upstream (Forks):

git remote add upstream [URL_ORIGINAL] git fetch upstream git merge upstream/main

🔒 Claves SSH

ssh-keygen -t rsa -b 4096 -C "email" ssh -T git@github.com # Testear conexión
