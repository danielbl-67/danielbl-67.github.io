⚠️ Troubleshooting Firebase: Seguridad y Despliegue

Problema / Error ¿Por qué pasa? Solución Limpia / Regla
Fallo de Permisos:
Missing or insufficient permissions		 La app intenta leer o escribir datos en Firestore pero chocas contra las reglas de seguridad por defecto (que bloquean todo en producción). Firestore Rules Configura las reglas en tu consola Cloud para permitir escritura solo a usuarios autenticados: allow read, write: if request.auth != null;
Fallo de Auth:
User token expired		 El cliente móvil intenta hacer una petición pero la sesión del token de Firebase ha caducado en segundo plano. FirebaseAuth Listener Implementa un escuchador dinámico en la app móvil (Android/Kotlin) para refrescar la sesión automáticamente: FirebaseAuth.getInstance().addAuthStateListener { auth -> ... }

📁 Cloud Firestore (Bases de datos NoSQL)

Estructura: A diferencia de SQL, aquí no hay filas ni tablas. Todo se organiza en **Colecciones** que contienen **Documentos** (similares a objetos JSON):

// Ejemplo de documento en la colección "entrenadores"
{
  "id": "USR_9823",
  "nombre": "Daniel Burdallo",
  "especialidad": "Nutrición",
  "activo": true
}

🔄 Datos en Tiempo Real (Realtime Listener)

El Súper Poder de Firebase: Sincronizar la app al instante sin necesidad de que el usuario refresque la pantalla de forma manual.

Cómo funciona: Creas un escuchador sobre la colección de Firestore que reacciona solo cuando hay cambios en el servidor:

db.collection("entrenadores")
  .addSnapshotListener { snapshot, e ->
    // Actualiza la interfaz del móvil al instante
  }

🔒 Autenticación Simplificada

Ventaja: Firebase gestiona de forma segura el cifrado de contraseñas y las sesiones de usuario en la nube, ahorrándote programar un servidor completo de logins.

Flujo estándar: Comprobar si el usuario ya está logueado al abrir la app móvil:

val user = Firebase.auth.currentUser
if (user != null) {
  // Saltar directo a la pantalla principal de GYMZY
}
