📡 Métodos HTTP (Verbos Semánticos)

Cada acción sobre tu base de datos debe mapearse con el método HTTP correcto:

  • GET: Recuperar información. GET /api/usuarios/42
  • POST: Crear un nuevo registro. El objeto viaja oculto en el cuerpo (Body) en formato JSON.
  • PUT: Actualizar un registro existente por completo.
  • DELETE: Eliminar un recurso. DELETE /api/pedidos/10

✅ Respuestas Correctas (Familias 2xx)

Cuando todo va sobre ruedas, tu API debe responder con precisión:

  • 200 OK: La petición ha sido un éxito (ideal para peticiones GET y PUT).
  • 201 Created: El recurso se ha creado correctamente en la base de datos (perfecto para respuestas POST tras registrar un usuario).
  • 204 No Content: Todo ha ido bien, pero la respuesta no devuelve texto (muy común al hacer un DELETE con éxito).

❌ Errores del Cliente (Familias 4xx)

El cliente (App móvil o web) ha enviado algo mal en la petición:

  • 400 Bad Request: Los datos enviados no cumplen con el formato o validaciones (ej: contraseña demasiado corta).
  • 404 Not Found: La URL no existe o el ID solicitado no se encuentra en la base de datos.
